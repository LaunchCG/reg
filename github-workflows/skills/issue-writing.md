# Issue Writing Skill

Expert guidance for creating clear, actionable GitHub issues.

## Core Principles

1. **Descriptive Title**: Clear summary of the issue
2. **Problem Statement**: What is the issue or feature request?
3. **Expected vs Actual**: For bugs, describe expected and actual behavior
4. **Reproduction Steps**: How to reproduce the issue
5. **Context**: Environment, versions, relevant details

## Issue Template Structure

### Bug Report
```markdown
## Description
Clear description of the bug

## Steps to Reproduce
1. Step one
2. Step two
3. Step three

## Expected Behavior
What should happen

## Actual Behavior
What actually happens

## Environment
- OS:
- Browser/Node version:
- App version:

## Additional Context
Screenshots, logs, or other relevant information
```

### Feature Request
```markdown
## Problem Statement
What problem does this solve?

## Proposed Solution
How should this work?

## Alternatives Considered
Other approaches considered

## Additional Context
Mockups, examples from other tools, etc.
```

## Best Practices

- Use labels to categorize issues (bug, enhancement, documentation)
- Assign to appropriate team members or milestones
- Link to related issues or PRs
- Include error messages and stack traces for bugs
- Provide minimal reproducible examples

## Anti-Patterns

- ❌ Vague titles like "It doesn't work"
- ❌ Missing reproduction steps for bugs
- ❌ No context or environment information
- ❌ Combining multiple unrelated issues
- ❌ Feature requests without explaining the problem
