# CLAUDE.md

This is a **dex package registry** hosted on Azure Blob Storage. It contains 19 packages for AI-augmented SDLC workflows, code quality tooling, and framework support.

## Key Commands

```bash
# Deploy infrastructure (Azure Blob Storage)
./infrastructure/deploy.sh

# Package all packages and deploy to registry
./deploy.sh

# Deploy infra + packages in one step
./deploy.sh --infra

# Clean build and redeploy
./deploy.sh --clean

# Package only (no deploy)
./scripts/package.sh

# Rebuild registry from Azure blobs (recovery)
python3 scripts/rebuild-registry-from-azure.py <storage-account-name>
```

## Package Structure Convention

Each package is a directory at the repo root containing:

```
<package-name>/
├── package.hcl          # Required: package manifest
├── skills/              # Skill definitions (SKILL.md + resources/)
│   └── <skill-name>/
│       ├── SKILL.md
│       └── resources/   # Templates, examples, reference docs
├── agents/              # Agent definitions (.md files)
├── commands/            # Command definitions (.md files)
├── rules/               # Rule definitions (.md files)
└── tasks.yaml           # Optional: MCP task automation
```

### package.hcl Format

Every `package.hcl` must include:
- `package` block with `name`, `version`, `description`, `platforms`
- Dual `claude_*` and `copilot_*` blocks for cross-platform support
- `dependency` blocks for package dependencies
- `file` blocks inside skills for resource files

Example:
```hcl
package {
  name        = "my-package"
  version     = "0.1.0"
  description = "Description here"
  platforms   = ["claude-code", "github-copilot"]
}

dependency "base-dev" {
  version = ">=0.1.0"
}

claude_skill "my-skill" {
  description = "Skill description"
  content     = file("skills/my-skill/SKILL.md")

  file {
    src  = "skills/my-skill/resources/template.md"
    dest = "resources/template.md"
  }
}

copilot_skill "my-skill" {
  description = "Skill description"
  content     = file("skills/my-skill/SKILL.md")
}
```

## Adding a New Package

1. Create a directory at the repo root: `mkdir my-package`
2. Create `my-package/package.hcl` with the format above
3. Add skills, agents, commands, rules as needed
4. Run `./deploy.sh` to build and publish

## Infrastructure

- **Azure Blob Storage** with static website hosting
- **Bicep** templates in `infrastructure/`
- `infrastructure/config.sh` is generated by deploy and contains `STORAGE_ACCOUNT`, `CONTAINER_NAME`, `REGISTRY_URL`
